# isr-blog の概要

Next.js で作成されたブログです。

管理画面より記事の投稿や削除が可能な ISR を採用。

記事・カテゴリ・画像の投稿・編集・削除タイミングで、Next.jsのサーバーアクション部分で

ページの再検証（revalidate）の実施を記述しています。

また、一定期間に応じた revalidate も行っています。

旅程表アプリとほぼ同時に作成をはじめており、「ほぼ初作成アプリ」でもあります。

下記 URL のブログ部分でこのアプリを使用しています。
URL：https://www.travel-memory-life.com/

## 使用している主な技術

- TypeScript
- Next.js 14
- node.js
- Tailwind CSS
- NextAuth
- PostgreSQL/supabase(DB/画像のストレージ)
- Prisma

## 主な機能

- 記事・カテゴリ・画像の CRUD
- 2 種類のデザインのカテゴリ
- カテゴリを SEO を意識したページに変更が可能
- 「記事・カテゴリ・画像」の「作成・編集・削除」でページをrevalidate
- 画像のバリデーション（mineタイプ / 拡張子 / サイズ ）
- 認証

## カテゴリを SEO を意識したページに変更可

カテゴリは２種類のデザインのページ作成ができます。

- カテゴリに属する記事一覧の表示
- SEO を意識したコンテンツの表示

投稿した公開設定の記事を元に、自動でリンクが作成されたページが作られます。

必要に応じて、タイトルの変更をしたり、コンテンツの挿入などを行い、カテゴリを 1 つのページとして作り込むことができます。

カテゴリページは「パンくずリスト」「サイドメニュー」「カテゴリに属する記事」からなど、内部リンクが集まりやすいページです。

そのため、「各記事へのリンク集」としてだけではなく、「記事ページとしての作成」もできるようなっています。

## revalidate の実施タイミング

各ページは一定期間ごとに revalidate を行う期間を設定しています。

また、「記事・カテゴリ・画像」が「追加・削除・編集」のタイミングで revalidate が行われるようにしています。

action フォルダ(app > action)に、ServerAction の項目が記載されているファイルに記載がされています。

## 画像のバリデーション

管理者のみが投稿できるブログですが、画像の拡張子と mine タイプをバリデーションしています。

また、ファイルのサイズは 1mb 以下に限定をしています。

サイズなど変更する場合は下記を変更してください。

components > lib > ValidateFile.ts
components > ui > FormImage.ts

## 認証について(Middleware)

認証は Nextauth を利用しています。

今後、私が自己制作していくアプリのテンプレートとして作成している背景もあり、

ブログとアプリの 2 種類での認証ができるようになっています。

未ログイン時のアクセスについては、「middleware.ts」でリダイレクトの制御をしています。

## ブログのログイン情報

ブログのログイン情報は「.env」 で設定をするようにしています。

項目名などは「.env.sample」を確認してください。

## セットアップ手順

1. リポジトリをクローン：

git clone https://github.com/haru0354/next-blog-isr

cd next-blog-isr

2. 依存関係をインストール：

npm install

3. 環境ファイルを用意：

cp .env.example .env

env ファイルへの記載

4. サーバーを起動：

npm run dev

### コマンドの概要

| コマンド      | 説明                                       |
| ------------- | ------------------------------------------ |
| `npm install` | フロントエンドの依存関係をインストール     |
| `npm run dev` | フロント・バックのサーバー起動と監視を開始 |

## clone する場合の注意事項

- 画像はフリーの素材の利用規約もありアップロードしていません。
- データベースは自分で用意してください。
- env.sample ファイルをご確認ください。

一部の画像ファイルは画像素材元の規約によるアップロードをしていません。
そのため、一部の画像が正常に表示されません。

「.env.sample」を「.env」に変更して、各項目への記載が必要となります。
