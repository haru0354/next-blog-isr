
# 概要

Next.jsで作成されたブログです。

管理画面より投稿や削除することでページが反映されるISRを採用。
revalidateは記事を投稿・編集することで、サーバー側の(action)で実施するタイミングを制御しています。

また、それとは別に一定期間でのrevalidateも行っています。

元々、作成したアプリの集客用に作成した管理者が投稿できるブログとなります。

## 画像

画像はフリーの素材を利用しており、利用規約の関係でアップロードはしていないです。

そのため、ヘッダー画像などそのままダウンロードしても、表示がされないようになっています。
(今後、サンプル画像を用意する予定です)

## 使用している主な技術

- Next.js 14
- node.js
- Tailwind CSS
- NextAuth
- prisma/supabase(DB/画像のストレージ)

## 新規追加できる項目

以下の３種の項目が追加できるようになっています。

- 記事
- カテゴリ
- 画像

## カテゴリはSEOを意識したページに変更可

カテゴリは２種類のページの作成ができます。

- カテゴリに属する記事一覧の表示
- SEOを意識したコンテンツの表示

カテゴリページに関してはパンくずリストやサイドメニューなど、基本的には内部リンクが集まりやすいページです。

そのため、記事と同じようにコンテンツを入れたページを作成できるようなっています。

ミドルワードなど、周辺知識が色々と必要なキーワードに適した、ページの作成が可能となっています。

カテゴリにコンテンツを入れなければ、記事一覧のみが表示するページとなり、コンテンツを入れれば、
コンテンツの下に記事一覧が自動で表示されるようになっています。

## revalidateの実施タイミング

revalidateは「記事・カテゴリ・画像」の３種が「追加・削除・編集」のタイミングで行われるようになっています。

actionフォルダ(app > action)に、ServerActionの項目が記載されているファイルがおいてあります。
また、各ページでも一定期間ごとにrevalidateを行う期間を設定しています。

## 画像のvalidate

管理者のみが投稿できるブログですが、画像の拡張子とmineタイプをバリデーションしています。

また、ファイルのサイズは1mb以下に限定をしています。

セキュリティを考慮して、ログイン情報が漏れたとしても、管理画面から画像以外のファイルをアップロードできないようにしました。

サイズなど変更する場合は下記を変更してください。

components > lib > ValidateFile.ts
components > ui > FormImage.ts

## 認証について(Middleware)

認証はNextauthを利用しています。

今後、自分が自己制作していくアプリのテンプレートとして作成している背景もあり、
ブログとアプリの2種類での認証ができるようになっています。

未ログイン時のアクセスについては、「middleware.ts」でリダイレクトの制御をしています。

## ブログのログイン情報

ブログのログイン情報は.envでの記載をするようにしています。

項目名などは「.env.sample」を確認してください。
利用する場合は記載をして.envに名前を変更してください。

## sitemap.xmlは自動で作成

sitemap.xmlに関しては記事の投稿をした際などに、自動で項目が追加されるように作成しています。

app > sitemap.ts

## 作成経緯

初の自己作成アプリとして、旅程表(旅のしおり)を作成できるアプリを作りました。

ユーザーを集める手段として、ブログでのSEOを使った集客をする為に、
ブログを作成しようと思ったのが始まりです。

パフォーマンス面だとSSGが良いとは見たのですが、Vercel でのデプロイをしたので、せっかくなのでISRでの構築を検討しました。(別個でSSGでのブログは作成しました)

個人的な過去の経験より、「良い、悪い、効果がある、効果がない」などネットの情報ではなく、
実際に自分自身で試してみる方が重要だと思っています。

そのため、実際にどの程度、パフォーマンスに違いがでるのかなど知りたかった面もあり、
かつ、せっかかく自己制作なので試したいものを作ろうと思い、Vercelにデプロイするならと作成しました。

SSGならサーバーの選択肢もあるので、別個で作成しようと思った経緯があります。

今後、自己制作していくアプリはブログでの集客をする予定なので、
ベースとなるテンプレートとして作成しようと、開発したのが始まりです。
(恐らくブログはSSGにしますが・・・)
